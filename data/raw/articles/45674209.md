# Django 6.0 beta 1 released

Posted by **Natalia Bidart** on Oct. 22, 2025

Django 6.0 beta 1 is now available. It represents the second stage in the 6.0 release cycle and is an opportunity to try out the changes coming in Django 6.0.

Django 6.0 assembles a mosaic of modern tools and thoughtful design, which you can read about in [the in-development 6.0 release notes](https://docs.djangoproject.com/en/dev/releases/6.0/).

Only bugs in new features and regressions from earlier Django versions will be fixed between now and the 6.0 final release. Translations will be updated following the "string freeze", which occurs when the release candidate is issued. The [current release schedule](https://www.djangoproject.com/download/6.0/roadmap/) calls for a release candidate in about a month, with the final release scheduled roughly two weeks later on December 3.

Early and frequent testing from the community will help minimize the number of bugs in the release. Updates on the release schedule are available [on the Django forum](https://forum.djangoproject.com/t/django-6-0-release-timeline-and-next-steps/41275).

As with all alpha and beta packages, this release is **not** for production use. However, if you'd like to try some of the new features or help find and fix bugs (which should be reported to [the issue tracker](https://code.djangoproject.com/newticket)), you can grab a copy of the beta package from [our downloads page](https://www.djangoproject.com/download/) or on PyPI.

The PGP key ID used for this release is Natalia Bidart: [2EE82A8D9470983E](https://github.com/nessita.gpg)
