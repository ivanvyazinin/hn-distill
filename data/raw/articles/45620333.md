# Forgejo v13.0 is available

[Forgejo v13.0](/download/) was released on 16 October 2025. You will find a short selection of the changes it introduces below and a [complete list in the release notes](https://codeberg.org/forgejo/forgejo/milestone/21377).

A [dedicated test instance](https://v13.next.forgejo.org/) is available to try it out. Before upgrading it is _strongly recommended_ to make a full backup as explained in the [upgrade guide](/docs/v13.0/admin/upgrade/) and carefully read _all breaking changes_ from the [release notes](https://codeberg.org/forgejo/forgejo/milestone/21377). If in doubt, do not hesitate to ask for help [in the chat room](https://matrix.to/#/#forgejo-chat:matrix.org).

## Summary[](#summary)

Content moderation is one of the main roadblocks for federation. A first step is now available to [users and admins](https://codeberg.org/forgejo/forgejo/pulls/6977). It was completed while ActivityPub related pull requests were [worked on in parallel and merged](https://codeberg.org/forgejo/forgejo/pulls?type=all&state=closed&labels=79349&milestone=21377).

Security was improved under the hood for [Forgejo Actions secrets](https://codeberg.org/forgejo/forgejo/pulls/8692) now using a more secure module [introduced in 2024](https://codeberg.org/forgejo/forgejo/pulls/5041) and [already in use for TOTP secrets](https://codeberg.org/forgejo/forgejo/pulls/6074) since Forgejo v10. In addition it is now possible to [require 2FA instance wide](https://codeberg.org/forgejo/forgejo/pulls/8753).

Forgejo Actions usability was improved with [access to all run attempts](https://codeberg.org/forgejo/forgejo/pulls/9017) of each workflow: they were stored but not available via the web UI. The workflow files are now [statically checked](https://forgejo.org/docs/v13.0/user/actions/troubleshooting/#schema-validation), for early detection of errors such as the usage of a wrong context in an expression like `${{ forgejo.server_url }}`.

### Reporting abusive content[](#reporting-abusive-content)

Moderation on public Forgejo instance is a hard and time-consuming task as there are no builtin tools to help with this. Although self-moderation allows users with no elevated privileges to [block abusive behavior](https://forgejo.org/docs/v13.0/user/blocking-user/), it is sometime necessary to get an instance administrator involved.

[It is now possible to report](https://codeberg.org/forgejo/forgejo/pulls/6977) users, organizations, repositories, issues, pull request or comments that should be looked at by instance administrators. A category and an explanation of why the content is problematic must be attached.

![Reporting abusive content](/_astro/2025-10-moderation-0.CNOZkc2u_rg4T4.webp)

Those reports are then [made available in the admin interface](https://codeberg.org/forgejo/forgejo/pulls/7905).

![Administrative interface for abusive content reports](/_astro/2025-10-moderation-1.95NidYUH_1l46Qf.webp)

When multiple reports are submitted for the same content, they are grouped and the counter from the right side shows how many open reports are linked to that content. Clicking on this counter will open a details page where the full details of all grouped reports can be viewed.

### Migration from Pagure[](#migration-from-pagure)

Repositories from [Pagure](https://pagure.io/) can be [migrated to Forgejo](https://codeberg.org/forgejo/forgejo/pulls/8513). It can be used, for instance, in the context of [the transition of Fedora from Pagure to Forgejo](https://fedoramagazine.org/fedora-moves-towards-forgejo-a-unified-decision/) that was decided in December 2024.

![Icon of pagure in the migration page](/_astro/2025-10-pagure.CgbBzWsI_22ENLX.webp)

### Configurable global 2FA enforcement[](#configurable-global-2fa-enforcement)

The [`[security].GLOBAL_TWO_FACTOR_REQUIREMENT`](https://forgejo.org/docs/v13.0/admin/config-cheat-sheet/#security-security) setting was [added](https://codeberg.org/forgejo/forgejo/pulls/8753) to require two factor authentication (TOTP, etc.) for either all users or instance administrators.

![Admin panel showing global 2FA enforcement](/_astro/2025-10-two-factor.C9A3CTJ3_ZccLD4.webp)

### Avatar image privacy[](#avatar-image-privacy)

Uploaded avatar images can sometimes contain unexpected metadata such as the location where the image was created, or the device the image was created with, stored in a format called EXIF. Forgejo now [removes EXIF data](https://codeberg.org/forgejo/forgejo/pulls/9638) when custom user and repository images are uploaded in order to reduce the risk of personally identifiable information being leaked unexpectedly.

A new [CLI subcommand](https://forgejo.org/docs/v13.0/admin/command-line/#doctor-avatar-strip-exif) `forgejo doctor avatar-strip-exif` can be used to strip EXIF information from all existing avatars; we recommend that administrators run this command once after upgrade in order to minimize this risk for existing stored files.

### View previous Forgejo Actions attempts[](#view-previous-forgejo-actions-attempts)

It is now [possible to view previous logs](https://codeberg.org/forgejo/forgejo/pulls/9017) for Actions runs that have been retried.

![Run attempt dropdown](/_astro/2025-10-run-attempt-dropdown.CH5_JQEJ_pAvq8.webp)

### Static check of Actions workflow files[](#static-check-of-actions-workflow-files)

[Forgejo Actions workflows](https://forgejo.org/docs/v13.0/user/actions/basic-concepts/#basic-syntax) are statically verified in the web UI for common errors such as using an incorrect context (e.g. `${{ badcontext.FORGEJO_REPOSITORY }}`) or a typo in a required keyword (e.g. `ruins-on:` instead of `runs-on:`). It is reported in the action page and the web page that displays the file in the repository.

This is in addition to similar tooling provided by [the Forgejo runner itself](https://code.forgejo.org/forgejo/runner/releases/tag/v9.0.0) and helps detect errors as soon as possible.

![Workflow validation error example](/_astro/2025-10-workflow-validation._k835nB6_Z4pTxw.webp)

### Show CI status on force push[](#show-ci-status-on-force-push)

When force-pushing to a pull request, a line is added to the conversation view to compare the two. However, the CI status of commits in such line was hidden behind an additional click. It is [now displayed next to the SHA of each commit](https://codeberg.org/forgejo/forgejo/pulls/8655), similar to commits from normal pushes.

![CI status in the pull request conversation page](/_astro/2025-10-status-force-push.4URhMI_A_xJtFE.webp)

### Markdown editor bold & italic keyboard shortcuts[](#markdown-editor-bold--italic-keyboard-shortcuts)

[Keyboard shortcuts were implemented](https://codeberg.org/forgejo/forgejo/pulls/9110) in the markdown editor for formatting features **Bold** and **Italic** so that they can be used more quickly. Behavior of using shortcuts is the same as clicking the buttons.

![Tooltip showing how to use keyboard shortcuts for bold and italics](/_astro/2025-10-shortcut.aJxvD4jX_zJ5qO.webp)

### Timestamps on release attachments[](#timestamps-on-release-attachments)

The time when a release attachment was uploaded is [now shown among other information about it](https://codeberg.org/forgejo/forgejo/pulls/8757).

![Timestamp tooltip on a release attachment](/_astro/2025-10-release-timestamp.BZii6Y-x_HauKb.webp)

### Tags are shown in commit lists[](#tags-are-shown-in-commit-lists)

When viewing repository’s commits, [tags are now shown next to the commits](https://codeberg.org/forgejo/forgejo/pulls/8759) they are associated with.

![Tags shown in the list of commits](/_astro/2025-10-commit-list-tag.ChsHTigu_YrTE.webp)

### Logger mode settings `LOGGER_<NAME>_MODE`[](#logger-mode-settings-logger_name_mode)

The [logger settings](https://forgejo.org/docs/v13.0/admin/config-cheat-sheet/#log-log) were [renamed](https://codeberg.org/forgejo/forgejo/pulls/8667) from `logger.<name>.MODE` to `LOGGER_<NAME>_MODE` to help setting them with environment variables when using a container image. For instance:

```
docker run --rm -e FORGEJO__log__LOGGER_ACCESS_MODE=file ... codeberg.org/forgejo/forgejo:13
```

Configuration files using `logger.<name>.MODE` are still supported but they are no longer documented. This change is specially useful when the name of the setting goes through various transformations that convert lowercase to uppercase, such as the [Forgejo helm chart](https://code.forgejo.org/forgejo-helm/forgejo-helm).

### 13.0-test daily releases[](#130-test-daily-releases)

Releases are built daily from the latest changes found in the [v13.0/forgejo](https://codeberg.org/forgejo/forgejo/src/branch/v13.0/forgejo) development branch. They are deployed to the [https://v13.next.forgejo.org](https://v13.next.forgejo.org) instance for manual verification in case a bug fix is of particular interest ahead of the next patch release. It can also be installed locally with:

*   OCI images: [root](https://codeberg.org/forgejo-experimental/-/packages/container/forgejo/13.0-test) and [rootless](https://codeberg.org/forgejo-experimental/-/packages/container/forgejo/13.0-test-rootless)
*   [Binaries](https://codeberg.org/forgejo-experimental/forgejo/releases/tag/v13.0-test)

Their names are staying the same but they are replaced by new builds every day.

## Get Forgejo v13.0[](#get-forgejo-v130)

See the [download page](/download/) for instructions on how to install Forgejo, and read the [release notes](https://codeberg.org/forgejo/forgejo/milestone/21377) for more information.

### Upgrading[](#upgrading)

Carefully read the [breaking bug fixes](https://codeberg.org/forgejo/forgejo/milestone/21377) section of the release notes.

The actual upgrade process is as simple as replacing the binary or container image with the corresponding [Forgejo binary](https://codeberg.org/forgejo/forgejo/releases/tag/v13.0.0) or [container image](https://codeberg.org/forgejo/-/packages/container/forgejo/13.0.0). If you’re using the container images, you can use the [`13.0` tag](https://codeberg.org/forgejo/-/packages/container/forgejo/13.0) to stay up to date with the latest `13.0.Y` patch release automatically.

Make sure to check the [Forgejo upgrade documentation](/docs/v13.0/admin/upgrade) for recommendations on how to properly backup your instance before the upgrade.

### Contribute to Forgejo[](#contribute-to-forgejo)

If you have any feedback or suggestions for Forgejo do not hold back, it is also your project. Open an issue in [the issue tracker](https://codeberg.org/forgejo/forgejo/issues) for feature requests or bug reports, reach out [on the Fediverse](https://floss.social/@forgejo), or drop into [the Matrix space](https://matrix.to/#/#forgejo:matrix.org) ([main chat room](https://matrix.to/#/#forgejo-chat:matrix.org)) and say hi!

### Donate[](#donate)

Forgejo is proud to be [funded transparently](https://codeberg.org/forgejo/sustainability/). Additionally, it accept donations [through Liberapay](https://liberapay.com/forgejo). It is also possible to [donate to Codeberg e.V.](https://docs.codeberg.org/improving-codeberg/donate/) in case the Liberapay option does not work out for you, and part of the funding is used to [compensate for work on Forgejo](https://codeberg.org/forgejo/sustainability/#forgejo-resources-per-year).

However, the Liberapay team allows for money to go directly to developers without a round-trip to Codeberg. Additionally, Liberapay allows for a steady and reliable funding stream next to other options, a crucial aspect for the project. The distribution of funds through Liberapay is [transparently controlled using the decision-making process](https://codeberg.org/forgejo/governance/src/branch/main/TEAMS.md#liberapay-team-members), and Forgejo contributors are encouraged to consider applying to benefit from this funding opportunity.

Thank you for using Forgejo and considering a donation, in case your financial situation allows you to.
