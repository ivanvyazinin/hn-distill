# What is OpenBSD ?

OpenBSD is a free UNIX©\-like operating system.  
It has been forked from NetBSD in 1995.

The development process concentrates on portability and security.  
There are about 10+ supported hardware architecture.

Some of the software (OpenSSH, LibreSSL, pf…) developed for OpenBSD are widely used in other free and proprietary software.

The official OpenBSD website is [https://www.openbsd.org](https://www.openbsd.org/) .

* * *

# Why using OpenBSD? _Because you can!_

The code is open and the sources freely available: [OpenBSD CVSweb](https://cvsweb.openbsd.org/src/) .

The installation media can be freely downloaded from [OpenBSD CDN](https://cdn.openbsd.org/pub/OpenBSD/) .

The system works on various hardware platforms: [OpenBSD supported platform list](https://www.openbsd.org/plat.html) .

The system is compatible with most virtualisation platforms: Qemu, KVM, Virtualbox, VMware, bhyve, vmd…

# Why using OpenBSD? _Because it is featurefull!_

It is developed and maintained with [security in mind](https://www.openbsd.org/security.html) .  
It ships with loads of [security innovations](https://www.openbsd.org/innovations.html) : privilege separation and revocation, stack protector, randomization,  
[pledge(2)](https://man.openbsd.org/pledge.2) , [unveil(2)](https://man.openbsd.org/unveil.2) , etc…

Here are a few server-based options:

*   Build a firewall with [pf(4)](https://man.openbsd.org/man4/pf.4) .
*   Build a router with [bgpd(8)](https://man.openbsd.org/man8/bgpd.8) .
*   Build a bastion with [sshd(8)](https://man.openbsd.org/sshd.8) .
*   Build a DHCP and/or DNS server with [dhcpd(8)](https://man.openbsd.org/dhcpd.8) , [nsd(8)](https://man.openbsd.org/nsd.8) , [unbound(8)](https://man.openbsd.org/unbound.8) .
*   Build a user directory with [ldapd(8)](https://man.openbsd.org/ldapd.8) .
*   Build a Web server with [httpd(8)](https://man.openbsd.org/httpd.8) .  
    Build a reverse-proxy with [relayd(8)](https://man.openbsd.org/relayd.8) .
*   Build a mail server with [smtpd(8)](https://man.openbsd.org/smtpd.8) and/or [spamd(8)](https://man.openbsd.org/spamd.8) .
*   Obtain and update TLS certificates from Let’s Encrypt using [acme-client(1)](https://man.openbsd.org/acme-client.1) .

It can also be used as a workstation thanks to integrated software like:

*   The [xenodm(1)](https://man.openbsd.org/man1/xenodm.1) connection manager.
*   The [Xorg(1)](https://man.openbsd.org/man1/Xorg.1) graphical server.
*   One of those window managers: [fvwm(1)](https://man.openbsd.org/man1/fvwm.1) , [cwm(1)](https://man.openbsd.org/man1/cwm.1) , [twm(1)](https://man.openbsd.org/twm.1) .
*   The [xterm(1)](https://man.openbsd.org/xterm.1) terminal emulator.
*   The [tmux(1)](https://man.openbsd.org/tmux.1) terminal multiplexer.
*   One of the classical UNIX editors and shells: [ed(1)](https://man.openbsd.org/ed.1) , [vi(1)](https://man.openbsd.org/vi.1) , [mg(1)](https://man.openbsd.org/mg.1) , [ksh(1)](https://man.openbsd.org/ksh.1) , [csh(1)](https://man.openbsd.org/csh.1) .

And many more…

# Why using OpenBSD? _Because it provides binary packages!_

A whole set of Free and OpenSource Software is available as binary packages for every supported architecture.

*   amd64: ~12000
*   arm64: ~12000
*   powerpc: ~8000
*   risc64: ~10000
*   sparc64: ~9000
*   etc …

Server-oriented highlighted software for OpenBSD 7.7 include Apache, Asterisk, Go, OpenJDK, MariaDB, Node.js, OpenLDAP, PHP, Postfix, PostgreSQL, Ruby, Rust, …

Workstation-oriented highlighted software for OpenBSD 7.7 include Chromium, ffmpeg, Gnome, KDE, Krita, LibreOffice, Mozilla Firefox, Mozilla Thunderbird, (Neo)Mutt, Python, Shotcut, TeX, (neo)vim, Xfce, …

# Why using OpenBSD? _Because it is simple!_

The text-mode installation wizard works the same way on the various architecture and connection type (`COM0`, `VGA`, `HDMI`, `KVM-IP`…).

The documentation [`man(1)`](https://man.openbsd.org/man) is complete and a welcome message pointing to [`afterboot(8)`](https://man.openbsd.org/afterboot) helps you start with using OpenBSD. The overall filesystem organisation is described in [`hier(7)`](https://man.openbsd.org/hier) .

Service management configuration is locate in `/etc/rc.conf.local`.  
Service management is available using a single [`rcctl(8)`](https://man.openbsd.org/rcctl) command.

The OS material is installed throughout `/`.  
Third-party material is installed under `/usr/local`.

The system configuration examples are located inside `/etc/examples/`.  
All software configuration defaults to `/etc/`.

The dedicated [`fw_update(8)`](https://man.openbsd.org/man8/fw_update.8) command manages non-free firmwares.

Binary packages management includes dealing with dependencies and [options](https://man.openbsd.org/ports#FLAVORS) .  
Three main commands deal with package management: [`pkg_info(1)`](https://man.openbsd.org/man1/pkg_info.1) , [`pkg_add(1)`](https://man.openbsd.org/pkg_add.1) , [`pkg_delete(1)`](https://man.openbsd.org/pkg_delete.1) .

Security patches are managed using the [`syspatch(8)`](https://man.openbsd.org/syspatch.8) command.  
System upgrades are driven by the [`sysupgrade(8)`](https://man.openbsd.org/sysupgrade.8) commands.
