# [Update on my Racket exit](https://blog.winny.tech/posts/update-on-my-racket-exit/)

Updated Wednesday, Apr 30, 2025

[Racket](https://blog.winny.tech/tags/racket/)

![Figure 1: Blasting off from planet Racket to other planets. Thanks ChatGPT.](/ox-hugo/racket-rocket-update.png)

Figure 1: Blasting off from planet Racket to other planets. Thanks ChatGPT.

It’s been almost two years since I [took a public step away from the Racket diaspora](/posts/racket-frustrates-me/). I’m writing this post to air out how my Racket projects have gone since then. I believe, as of now, I have zero Racket codebases to maintain — hooray!

I want to show how to handle old codebases. Let’s reaffirm the importance of handing off inactive projects instead of holding on to them. Kill your darlings. Let your babies fly from the nest. Let go of your projects so they can evolve without you. Embrace growth, even if it comes with a modicum of chaos. That’s the path. Something beautiful will grow.

## [§](#steps-to-hand-off-a-racket-project)Steps to hand off a Racket project

Here’s my checklist for handing off a Racket project. _Only step 4 is Racket specific._

1.  Find an interested future maintainer.
2.  Create a GitHub org and transfer the repository into the org.
3.  Add your future maintainer as a GitHub organization owner.
4.  Update [pkgs.racket-lang.org](https://pkgs.racket-lang.org/) entries with
    1.  An additional _author email_ linked to the future maintainer’s pkgs.racket-lang.org account
    2.  Update the repository path because presumably, GitHub will eventually stop redirecting after a repository move.
5.  Communicate and clarify any further questions for the new maintainer as they arise. Or not and disappear. Long term projects success hinges on providing the easiest possible hand-off and continued support, as needed.

## [§](#toml-racket-was-rehomed)toml-racket was rehomed

Turns out, more people used [my revival](https://github.com/toml-racket/toml-racket) of [Greg](https://www.greghendershott.com/)’s toml-racket[1](#fn:1) than just me. Go figure: TOML is sweet and simple. The revival codebase touts combined encoding and decoding support.

Thanks to [Benjamin Yeung](https://github.com/bzyeung) for adopting toml-racket. 🙇

## [§](#tinybasic-dot-rkt-was-rehomed)tinybasic.rkt was rehomed

_[Previously](/posts/lang-tinybasic/)._

I was on the fence with archiving the [tinybasic.rkt](https://github.com/tinybasic-rkt/tinybasic-rkt) repository; given the 10+ stars for this boutique programming language that exists within a niche community, it seemed like a raw deal for these passionate individuals.

A random GitHub Pull Request appeared! I’m not set up anymore to QA and review Racket related patches so it made sense to offer up the project to interested users. I created an issue asking for volunteers to take over the project. Thankfully _the s-expressions aligned_ and a maintainer appeared!

Big thanks to [Jörgen Brandt](https://github.com/joergen7) for taking on maintainership of tinybasic. 🙇

## [§](#other-racket-packages-removed-from-pkgs-dot-racket-lang-dot-org)Other Racket packages removed from pkgs.racket-lang.org

The other packages seemed to have no users besides me. Since I was no longer using the packages, I deleted them from the Racket Catalog and then archived the GitHub repositories.

The removed packages include (though, I can’t recollect if this list is exhaustive or not):

[4chdl](https://github.com/winny-/4chdl)

4chan media downloader. 4chan is dead anyways so this is not useful to anyone, anymore.

[ssh-hack](https://github.com/winny-/ssh-hack)

a way to play nethack online. I rewrote this tool in Python and it’s been easier to deploy.

[tool](https://github.com/winny-/tool)

a logging and general purpose CLI utility framework thingy.

## [§](#admiring-old-darlings-experiences)Admiring old darlings, experiences

It hit me just how much code I’d written in Racket — tons of it. [_A lot._](https://github.com/winny-?language=racket&q=&sort=&tab=repositories) [_Loads of stuff._](https://gist.github.com/search?q=language%3Aracket+%40winny-&ref=searchresults)

The first Racket code that I can find is from circa 2015. It was [a command line tool to upload text to the sprunge.us PasteBin website](https://gist.github.com/winny-/73a4e5504a645178a990). It was already pretty decent code honestly. Passable as any beginner dev in the community.

And here’s a collection of my favorite Racket experiments:

1.  Language implementations - [Subby](https://github.com/winny-/subby), [TinyBasic](https://github.com/tinybasic-rkt/tinybasic-rkt), [brainfuck](https://gist.github.com/winny-/d7ba8763b51e6bc2150284d6a8322b6d), [a doctest language](https://gitlab.com/winny-archive/zprava/random_files/-/tree/master/documentation/doctest?ref_type=heads) embedded [in Markdown](https://gitlab.com/winny-archive/zprava/random_files/-/blob/master/documentation/API.md?ref_type=heads) used to test API calls.
2.  Network protocol support to query [Minecraft](https://github.com/winny-/minecraft-ping), [ET:QW](https://github.com/winny-/etqw-ping-racket), [RCON-enabled](https://github.com/winny-/rcon-racket) game servers, [Mumble](https://github.com/winny-/mumble-ping) VoIP servers, so on
3.  [Using](https://github.com/search?q=repo%3Awinny-%2Faoc%20language%3ARacket%20&type=code) Racket to [excel](https://github.com/search?q=repo%3Awinny-%2Fpractice%20language%3ARacket%20&type=code) coding competitions . _It’s a fun party trick, but won’t land you a job, by the way._

_It’s not exhaustive._ I didn’t list off any GUI, Web, IRC, command line utility things, or anything with excessive macro use. Those activities were seldom fun — too tedious. Especially anything web related.

[Bogdanp](https://defn.io/) has been working diligently on streamlining the web story, so it’s been a lot nicer with his batteries-included Racket APIs.

## [§](#fin)Fin

![Figure 2: On to the next thing. Thanks ChatGPT.](/ox-hugo/rocket-through-space.png)

Figure 2: On to the next thing. Thanks ChatGPT.

It’s lovely to be done with these projects and redirecting focus to the next endeavor.

I’ve been writing a great deal of Python, Bash, Awk, Perl 5 for my own consumption and it’s kept my focus free for other activity such as biking or cooking.

For another time: let’s talk about incentives in software development. This is the cornerstone of anything coding, not the language. Onboard, encourage, and reward, or not, at your project’s peril.

_If you don’t have community (including yourself) engaged, the code is effectively dead no matter the language._
