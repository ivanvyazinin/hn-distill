## Identity Types

Posted by Bartosz Milewski under [Category Theory](https://bartoszmilewski.com/category/category-theory/), [Homotopy](https://bartoszmilewski.com/category/homotopy/), [Homotopy Type Theory](https://bartoszmilewski.com/category/homotopy-type-theory/) | Tags: [Category Theory](https://bartoszmilewski.com/tag/category-theory/), [Homotopy](https://bartoszmilewski.com/tag/homotopy/), [HoTT](https://bartoszmilewski.com/tag/hott/), [Identity Type](https://bartoszmilewski.com/tag/identity-type/) |

Previously: [Models of (Dependent) Type Theory](https://bartoszmilewski.com/2025/09/05/models-of-dependent-type-theory/).

There is a deep connection between mathematics and programming. Computer programs deal with such mathematical objects as numbers, vectors, monoids, functors, algebras, and many more. We can implement such structures in most programming languages. For instance, here’s the definition of natural numbers in Haskell:

```
data Nat where
  Z :: Nat           -- zero
  S :: Nat -> Nat    -- successor
```

There is a problem, though, with encoding the laws that they are supposed to obey. These laws are expressed using equalities. But not all equalities are created equal. Take, for instance, this definition of addition:

```
plus :: Nat -> Nat -> Nat
plus Z n = n
plus (S n) m = S (plus n m)
```

The first clause tells us that ![0 + n](https://s0.wp.com/latex.php?latex=0+%2B+n&bg=ffffff&fg=29303b&s=0&c=20201002) is equal to ![n](https://s0.wp.com/latex.php?latex=n&bg=ffffff&fg=29303b&s=0&c=20201002). This is called _definitional equality_, since it’s just part of the definition. However, if we reverse the order of the addends and try to prove ![n + 0 = n](https://s0.wp.com/latex.php?latex=n+%2B+0+%3D+n&bg=ffffff&fg=29303b&s=0&c=20201002), there is no obvious shortcut. We have to prove it the hard way! This second type of equality is called _propositional_.

In traditional mathematics equality is treated as a relation. Two things are either equal or not. In type theory, on the other hand, equality is a type. It’s a type of proofs of equality. When you postulate that two things are equal, you specify the _type_ of proofs that you are willing to accept.

Equality is a dependent type, because it depends on the _values_ that are being compared. For a pair of values of the type ![A](https://s0.wp.com/latex.php?latex=A&bg=ffffff&fg=29303b&s=0&c=20201002), the identity (or equality) type is denoted by ![\text{Id}_A](https://s0.wp.com/latex.php?latex=%5Ctext%7BId%7D_A&bg=ffffff&fg=29303b&s=0&c=20201002):

![\frac{a, b \colon A}{\text{Id}_A (a, b) \colon \text{Type}}](https://s0.wp.com/latex.php?latex=%5Cfrac%7Ba%2C+b+%5Ccolon+A%7D%7B%5Ctext%7BId%7D_A+%28a%2C+b%29+%5Ccolon+%5Ctext%7BType%7D%7D&bg=ffffff&fg=29303b&s=0&c=20201002)

(In this notation, assumptions are listed above the horizontal line.)

You’ll also see ![\text{Id}_A(a, b)](https://s0.wp.com/latex.php?latex=%5Ctext%7BId%7D_A%28a%2C+b%29&bg=ffffff&fg=29303b&s=0&c=20201002) written as ![a =_A b](https://s0.wp.com/latex.php?latex=a+%3D_A+b&bg=ffffff&fg=29303b&s=0&c=20201002), or even ![a = b](https://s0.wp.com/latex.php?latex=a+%3D+b&bg=ffffff&fg=29303b&s=0&c=20201002), if the type ![A](https://s0.wp.com/latex.php?latex=A&bg=ffffff&fg=29303b&s=0&c=20201002) is obvious from the context.

When two values are not equal, this type is uninhabited. Otherwise it’s inhabited by proofs, or witnesses, of equality. This is consistent with the _propositions as types_ interpretation of type theory.

For the identity type to be useful, we have to have a way to populate it with values — proofs of equality. We do that by providing an introduction rule:

![\frac{a \colon A}{\text{refl}(a) \colon \text{Id}_A(a, a)}](https://s0.wp.com/latex.php?latex=%5Cfrac%7Ba+%5Ccolon+A%7D%7B%5Ctext%7Brefl%7D%28a%29+%5Ccolon+%5Ctext%7BId%7D_A%28a%2C+a%29%7D&bg=ffffff&fg=29303b&s=0&c=20201002)

The constructor ![\text{refl}](https://s0.wp.com/latex.php?latex=%5Ctext%7Brefl%7D&bg=ffffff&fg=29303b&s=0&c=20201002) generates a witness to the obvious fact that, for every ![a](https://s0.wp.com/latex.php?latex=a&bg=ffffff&fg=29303b&s=0&c=20201002), ![a = a](https://s0.wp.com/latex.php?latex=a+%3D+a&bg=ffffff&fg=29303b&s=0&c=20201002). Here, ![\text{refl}](https://s0.wp.com/latex.php?latex=%5Ctext%7Brefl%7D&bg=ffffff&fg=29303b&s=0&c=20201002) is the abbreviation of _reflexivity_.

At first sight it doesn’t seem like we have gained anything by turning equality into a type. Indeed, if ![\text{refl}](https://s0.wp.com/latex.php?latex=%5Ctext%7Brefl%7D&bg=ffffff&fg=29303b&s=0&c=20201002) were the only inhabitant of ![\text{Id}_A](https://s0.wp.com/latex.php?latex=%5Ctext%7BId%7D_A&bg=ffffff&fg=29303b&s=0&c=20201002), we’d just buy ourselves a lot of headaches for nothing. The trouble is that, if there is a type ![\text{Id}_A](https://s0.wp.com/latex.php?latex=%5Ctext%7BId%7D_A&bg=ffffff&fg=29303b&s=0&c=20201002), then there’s also a type ![\text{Id}_{\text{Id}_A}](https://s0.wp.com/latex.php?latex=%5Ctext%7BId%7D_%7B%5Ctext%7BId%7D_A%7D&bg=ffffff&fg=29303b&s=0&c=20201002) (type of equality of proofs of equality), and so on, ad infinitum. This used to be a bane of Martin Löf type theory, but it became a bounty for Homotopy Type Theory. So let’s imagine that the identity type may have non-trivial inhabitants. We’ll call these inhabitants _paths_. The trivial paths generated by ![\text{refl}](https://s0.wp.com/latex.php?latex=%5Ctext%7Brefl%7D&bg=ffffff&fg=29303b&s=0&c=20201002) are degenerate do-nothing loops.

But first, let’s consider the identity _elimination rule_: the mapping out of the identity type. What data do we need to provide in order to define a function from a path ![p \colon \text{Id}_A(x, y)](https://s0.wp.com/latex.php?latex=p+%5Ccolon+%5Ctext%7BId%7D_A%28x%2C+y%29&bg=ffffff&fg=29303b&s=0&c=20201002) to some type ![D](https://s0.wp.com/latex.php?latex=D&bg=ffffff&fg=29303b&s=0&c=20201002)? In full generality, ![D](https://s0.wp.com/latex.php?latex=D&bg=ffffff&fg=29303b&s=0&c=20201002) should be a dependent type family parameterized by the path ![p](https://s0.wp.com/latex.php?latex=p&bg=ffffff&fg=29303b&s=0&c=20201002) as well as its endpoints ![x](https://s0.wp.com/latex.php?latex=x&bg=ffffff&fg=29303b&s=0&c=20201002) and ![y](https://s0.wp.com/latex.php?latex=y&bg=ffffff&fg=29303b&s=0&c=20201002):

![x, y \colon A, \; p \colon \text{Id}_A (x, y) \vdash D(x, y, p) \colon \text{Type}](https://s0.wp.com/latex.php?latex=x%2C+y+%5Ccolon+A%2C+%5C%3B+p+%5Ccolon+%5Ctext%7BId%7D_A+%28x%2C+y%29+%5Cvdash+D%28x%2C+y%2C+p%29+%5Ccolon+%5Ctext%7BType%7D&bg=ffffff&fg=29303b&s=0&c=20201002)

Our goal is to define a ![D](https://s0.wp.com/latex.php?latex=D&bg=ffffff&fg=29303b&s=0&c=20201002)\-valued mapping ![J](https://s0.wp.com/latex.php?latex=J&bg=ffffff&fg=29303b&s=0&c=20201002) that works for any path in ![A](https://s0.wp.com/latex.php?latex=A&bg=ffffff&fg=29303b&s=0&c=20201002):

![x, y \colon A, \; p \colon \text{Id}_A(x, y) \vdash J(x, y, p) \colon D(x, y, p)](https://s0.wp.com/latex.php?latex=x%2C+y+%5Ccolon+A%2C+%5C%3B+p+%5Ccolon+%5Ctext%7BId%7D_A%28x%2C+y%29+%5Cvdash+J%28x%2C+y%2C+p%29+%5Ccolon+D%28x%2C+y%2C+p%29&bg=ffffff&fg=29303b&s=0&c=20201002)

The trick is to realize that there is only one family of constructors of the identity type, ![\text{refl}(a)](https://s0.wp.com/latex.php?latex=%5Ctext%7Brefl%7D%28a%29&bg=ffffff&fg=29303b&s=0&c=20201002), so it’s enough to specify how ![J](https://s0.wp.com/latex.php?latex=J&bg=ffffff&fg=29303b&s=0&c=20201002) is supposed to act on it. We do this by providing a dependent function ![d](https://s0.wp.com/latex.php?latex=d&bg=ffffff&fg=29303b&s=0&c=20201002) (a dependent function is a function whose return type depends on the _value_ of its argument):

![a \colon A \vdash d(a) \colon D(a, a, \text{refl}(a))](https://s0.wp.com/latex.php?latex=a+%5Ccolon+A+%5Cvdash+d%28a%29+%5Ccolon+D%28a%2C+a%2C+%5Ctext%7Brefl%7D%28a%29%29&bg=ffffff&fg=29303b&s=0&c=20201002)

This is very similar to constructing a mapping out of a Boolean by specifying its action on the two constructors, ![\text{True}](https://s0.wp.com/latex.php?latex=%5Ctext%7BTrue%7D&bg=ffffff&fg=29303b&s=0&c=20201002) and ![\text{False}](https://s0.wp.com/latex.php?latex=%5Ctext%7BFalse%7D&bg=ffffff&fg=29303b&s=0&c=20201002). Except that here we have a whole family of constructors ![\text{refl}(a)](https://s0.wp.com/latex.php?latex=%5Ctext%7Brefl%7D%28a%29&bg=ffffff&fg=29303b&s=0&c=20201002) parameterized by ![a](https://s0.wp.com/latex.php?latex=a&bg=ffffff&fg=29303b&s=0&c=20201002).

This is enough information to uniquely determine the action of ![J](https://s0.wp.com/latex.php?latex=J&bg=ffffff&fg=29303b&s=0&c=20201002) on any path, provided it agrees with ![d](https://s0.wp.com/latex.php?latex=d&bg=ffffff&fg=29303b&s=0&c=20201002) along ![\text{refl}](https://s0.wp.com/latex.php?latex=%5Ctext%7Brefl%7D&bg=ffffff&fg=29303b&s=0&c=20201002):

![J(a, a, \text{refl}(a)) = d(a)](https://s0.wp.com/latex.php?latex=J%28a%2C+a%2C+%5Ctext%7Brefl%7D%28a%29%29+%3D+d%28a%29&bg=ffffff&fg=29303b&s=0&c=20201002)

This procedure for defining mappings out of identity types is called _path induction_.

## Categorical Model

We’ll build a categorical interpretation of identity types making as few assumptions as possible. We’ll model ![\text{Id}_A](https://s0.wp.com/latex.php?latex=%5Ctext%7BId%7D_A&bg=ffffff&fg=29303b&s=0&c=20201002) as a fibration:

![\text{Id}_A \xrightarrow{\pi} A \times A](https://s0.wp.com/latex.php?latex=%5Ctext%7BId%7D_A+%5Cxrightarrow%7B%5Cpi%7D+A+%5Ctimes+A&bg=ffffff&fg=29303b&s=0&c=20201002)

The introduction rule asserts the existence of the arrow:

![A \xrightarrow{\text{refl}} \text{Id}_A](https://s0.wp.com/latex.php?latex=A+%5Cxrightarrow%7B%5Ctext%7Brefl%7D%7D+%5Ctext%7BId%7D_A&bg=ffffff&fg=29303b&s=0&c=20201002)

For any given ![a \colon A](https://s0.wp.com/latex.php?latex=a+%5Ccolon+A&bg=ffffff&fg=29303b&s=0&c=20201002), this arrow selects a diagonal element of the identity type, ![\text{Id}_A(a, a)](https://s0.wp.com/latex.php?latex=%5Ctext%7BId%7D_A%28a%2C+a%29&bg=ffffff&fg=29303b&s=0&c=20201002). When projected down to ![A \times A](https://s0.wp.com/latex.php?latex=A+%5Ctimes+A&bg=ffffff&fg=29303b&s=0&c=20201002) using ![\pi](https://s0.wp.com/latex.php?latex=%5Cpi&bg=ffffff&fg=29303b&s=0&c=20201002), it lands on the diagonal of ![A \times A](https://s0.wp.com/latex.php?latex=A+%5Ctimes+A&bg=ffffff&fg=29303b&s=0&c=20201002). In a cartesian category, the diagonal is defined by the arrow ![\Delta \colon A \to A \times A](https://s0.wp.com/latex.php?latex=%5CDelta+%5Ccolon+A+%5Cto+A+%5Ctimes+A&bg=ffffff&fg=29303b&s=0&c=20201002), so we have:

![\Delta = \pi \circ \text{refl}](https://s0.wp.com/latex.php?latex=%5CDelta+%3D+%5Cpi+%5Ccirc+%5Ctext%7Brefl%7D&bg=ffffff&fg=29303b&s=0&c=20201002)

In other words, there is a _factorization_ of the diagonal arrow.

[![](https://bartoszmilewski.com/wp-content/uploads/2025/09/untitled_artwork-3.jpeg?w=645)](https://bartoszmilewski.com/wp-content/uploads/2025/09/untitled_artwork-3.jpeg)

The elimination rule starts with defining a dependent type ![D](https://s0.wp.com/latex.php?latex=D&bg=ffffff&fg=29303b&s=0&c=20201002), which can be modeled as a fibration ![\rho](https://s0.wp.com/latex.php?latex=%5Crho&bg=ffffff&fg=29303b&s=0&c=20201002):

![D \xrightarrow{\rho} \text{Id} \xrightarrow{\pi} A \times A](https://s0.wp.com/latex.php?latex=D+%5Cxrightarrow%7B%5Crho%7D+%5Ctext%7BId%7D+%5Cxrightarrow%7B%5Cpi%7D+A+%5Ctimes+A&bg=ffffff&fg=29303b&s=0&c=20201002)

We provide a dependent function ![d](https://s0.wp.com/latex.php?latex=d&bg=ffffff&fg=29303b&s=0&c=20201002), which is interpreted as an arrow ![d : A \to D](https://s0.wp.com/latex.php?latex=d+%3A+A+%5Cto+D&bg=ffffff&fg=29303b&s=0&c=20201002) that maps ![a](https://s0.wp.com/latex.php?latex=a&bg=ffffff&fg=29303b&s=0&c=20201002) to an element ![d(a)](https://s0.wp.com/latex.php?latex=d%28a%29&bg=ffffff&fg=29303b&s=0&c=20201002) of ![D(a, a, \text{refl}(a))](https://s0.wp.com/latex.php?latex=D%28a%2C+a%2C+%5Ctext%7Brefl%7D%28a%29%29&bg=ffffff&fg=29303b&s=0&c=20201002). So, when projected back to ![\text{Id}_A](https://s0.wp.com/latex.php?latex=%5Ctext%7BId%7D_A&bg=ffffff&fg=29303b&s=0&c=20201002) using ![\rho](https://s0.wp.com/latex.php?latex=%5Crho&bg=ffffff&fg=29303b&s=0&c=20201002), it produces the same element as ![\text{refl}(a)](https://s0.wp.com/latex.php?latex=%5Ctext%7Brefl%7D%28a%29&bg=ffffff&fg=29303b&s=0&c=20201002)

![\rho \circ d = \text{refl}](https://s0.wp.com/latex.php?latex=%5Crho+%5Ccirc+d+%3D+%5Ctext%7Brefl%7D&bg=ffffff&fg=29303b&s=0&c=20201002)

[![](https://bartoszmilewski.com/wp-content/uploads/2025/09/untitled_artwork-3-1.jpeg?w=567)](https://bartoszmilewski.com/wp-content/uploads/2025/09/untitled_artwork-3-1.jpeg)

Given these assumptions, there exists a mapping:

![x, y \colon A, p \colon \text{Id}_A(x, y) \vdash J(x, y, p) \colon D(x, y, p)](https://s0.wp.com/latex.php?latex=x%2C+y+%5Ccolon+A%2C+p+%5Ccolon+%5Ctext%7BId%7D_A%28x%2C+y%29+%5Cvdash+J%28x%2C+y%2C+p%29+%5Ccolon+D%28x%2C+y%2C+p%29&bg=ffffff&fg=29303b&s=0&c=20201002)

which is interpreted as a section ![\text{Id}_A \to D](https://s0.wp.com/latex.php?latex=%5Ctext%7BId%7D_A+%5Cto+D&bg=ffffff&fg=29303b&s=0&c=20201002). The section condition is:

![\rho \circ J = id_{\text{Id}_A}](https://s0.wp.com/latex.php?latex=%5Crho+%5Ccirc+J+%3D+id_%7B%5Ctext%7BId%7D_A%7D&bg=ffffff&fg=29303b&s=0&c=20201002)

This mapping is unique if we assume that it agrees with ![d](https://s0.wp.com/latex.php?latex=d&bg=ffffff&fg=29303b&s=0&c=20201002) when restricted to ![\text{refl}](https://s0.wp.com/latex.php?latex=%5Ctext%7Brefl%7D&bg=ffffff&fg=29303b&s=0&c=20201002):

![J \circ \text{refl} = d](https://s0.wp.com/latex.php?latex=J+%5Ccirc+%5Ctext%7Brefl%7D+%3D+d&bg=ffffff&fg=29303b&s=0&c=20201002)

Consequently, we can illustrate the path induction rule using a single commuting diagram:

[![](https://bartoszmilewski.com/wp-content/uploads/2025/09/untitled_artwork-3-2.jpeg?w=653)](https://bartoszmilewski.com/wp-content/uploads/2025/09/untitled_artwork-3-2.jpeg)

If the outer square commutes then there is a unique diagonal arrow that makes the two triangles commute. The similarity to the lifting property of homotopy theory is not accidental, as we’ll see next.
