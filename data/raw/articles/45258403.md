# [Infinite Mac: Resource Fork Roundtripping #](https://blog.persistent.info/2025/09/infinite-mac-resource-forks.html)

Date: Monday, September 15, 2025

Labels: [Infinite Mac](https://blog.persistent.info/search/label/Infinite%20Mac), [Retro](https://blog.persistent.info/search/label/Retro)

One of the frequent emails that I get about [Infinite Mac](https://infinitemac.org/) is “I’ve been copying over these files to every mac I’ve owned for the past 20 (or 30 or 40) years, how can I load them into your site?” While there’s been a “The Outside World” drive from [the very beginning](https://blog.persistent.info/2022/03/blog-post.html#the-outside-world), all it can do is a best-effort import/export using available browser APIs. Classic Mac OS used [resource forks](https://en.wikipedia.org/wiki/Creator_code) for structured data and stored [additional metadata](https://en.wikipedia.org/wiki/Creator_code) in the file system (where it [belongs](https://arstechnica.com/gadgets/2001/08/metadata/)). Modern macOS still [supports](https://stackoverflow.com/a/68741842) all of this, but unfortunately none of that data is visible to web apps - all they can read is the “data fork” of the file (i.e. its raw contents).

This is a long-standing issue – once Macintosh files started to be distributed over the Internet in the mid-90s, they were usually in a container format like [MacBinary](https://en.wikipedia.org/wiki/MacBinary) or [StuffIt](https://en.wikipedia.org/wiki/StuffIt#:~:text=Although%20Mac%20files,in%20macOS.) to preserve this extra data. While StuffIt is [still around](https://www.stuffit.com/), having to download a separate program goes against the ethos of Infinite Mac being as easy to use as possible. After some investigation, I remembered that .zip archives created by modern macOS [preserve resource forks](https://keith.github.io/xcode-man-pages/ditto.1.html#sequesterRsrc) (or even ancient Mac OS X - this feature [dates back to 10.3](https://cocoadev.github.io/BOMArchiveHelper/)). Specifically they create a parallel `__MACOSX` directory structure that has metadata and resource forks in an [AppleDouble](https://en.wikipedia.org/wiki/AppleSingle_and_AppleDouble_formats) file.

A while back I added [detection](https://github.com/mihaip/infinite-mac/commit/41c6023eee5473558b508c2c1d6efd052776c6a2) of such .zip files, which helped with getting data in. More recently, I also [switched](https://github.com/mihaip/infinite-mac/commit/4e2134d8557e4d14ff0822af43e1ed042d356c3d) the .zip files that Infinite Mac generates when exporting to also use this convention, allowing the Finder to expand and reconstruct the full contents. I also wanted to make this feature somewhat discoverable, and I discovered an exemption to the “browsers can’t see resource forks” limitation that I mentioned above – when dragging a whole folder into the window it’s possible to [read](https://github.com/mihaip/infinite-mac/commit/7a39dff8bc8d6df881044b3b5886bd9a894dba74) a file’s resource fork via a magic `_file_/..namedfork/rsrc` [path](https://en.wikipedia.org/wiki/Resource_fork#:~:text=In%20macOS%2C%20forks%20are%20named%20file/..namedfork/forkname%2C%20e.g.%2C%20the%20resource%20fork%20of%20the%20file%20IMG_0593.jpg%20is%20IMG_0593.jpg/..namedfork/rsrc.%20The%20ls%20command%20supports%20a%20%2Dl%40%20option%20which%20lists%20a%20file%27s%20forks.) (sadly there’s no equivalent for the metadata). I [used](https://github.com/mihaip/infinite-mac/commit/ea12604a5789501eb6dcc3e8480c4cbe3b5a2bd0) this to show a notification to remind users of the .zip capabilities.

This video shows these features working together - an old [app of mine](https://mscape.com/) starts out on the modern macOS host - it can be imported, its `vers` resource edited, and then exported back out (miraculous today’s Finder still displays information based on it).

### Odds and Ends

I also made the need to click into the window to trigger pointer lock for emulators that need it (DingsuPPC, PearPC, Previous) more [discoverable](https://github.com/mihaip/infinite-mac/commit/98717c0476a99fc1217294cf188b58baf3b12fc8) using the same notification mechanism. Thanks to [this tip](https://federated.saagarjha.com/objects/e9bcdd9e-9182-403b-a173-265ac5e89583) I figured out how to (ab)use GitHub releases to store large files, letting me finally [share the Mac OS X disk images](https://github.com/mihaip/infinite-mac/commit/a22674a922677e0c7f3f2b6d5877fd9ea68d0e01) I’ve been using. Finally I fixed some longstanding issues with [audio playback in some browsers](https://github.com/mihaip/infinite-mac/commit/630280c513560fc9d9f092b1c3daff220626563d) and [stuck modifier keys](https://github.com/mihaip/infinite-mac/commit/b0104f1050e6fe98aa73ca0680fdd96db2f10793).
