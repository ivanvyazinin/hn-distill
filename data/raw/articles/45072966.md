# Pearl

[](#pearl)

An Erlang lexer and syntax highlighter for Gleam!

[![Package Version](https://camo.githubusercontent.com/1f450038cfaf5f3d0bb727633e44ea334343ac49c99be28162f1ed8d2d72eac9/68747470733a2f2f696d672e736869656c64732e696f2f686578706d2f762f706561726c)](https://hex.pm/packages/pearl) [![Hex Docs](https://camo.githubusercontent.com/4bb6f0346c92c97533ee101eb333cf8ab3c7c7285aa966aed1ba529e055f398b/68747470733a2f2f696d672e736869656c64732e696f2f62616467652f6865782d646f63732d666661666633)](https://hexdocs.pm/pearl/)

Pearl is a lexer and syntax highlighter for Erlang, written in Gleam. The lexer is based on [`glexer`](https://hexdocs.pm/glexer) and [`just`](https://hexdocs.pm/just), allowing you to convert Erlang source code into tokens. There is also an API which allows you to highlight Erlang code using ansi colours, html or a custom format. Heavily inspired by [`contour`](https://hexdocs.pm/contour).

gleam add pearl@2

import pearl

pub fn main() {
  let code \= "
\-module(hello).
\-export(\[hello\_world/0\]).
hello\_world() -> io:fwrite(\\"Hello, world!\\\\n\\").
"

  let lexer \= pearl.new(code)
  // Lex syntax tokens for parsing or other uses
  let #(tokens, errors) \= pearl.tokenise(lexer)
  let assert \[\] \= errors
  parse\_erlang(tokens)

  // Highlight with ansi codes to print in the terminal
  let highlighted \= pearl.highlight\_ansi(code)
  io.println(highlighted)

  // Render to html to show in the browser
  let html \= pearl.highlight\_html(code)
  io.println("<pre><code>" <> html <> "</code></pre>")

  // Convert to "highlighting tokens" to highlight in some other way
  let highlight\_tokens \= pearl.highlight\_tokens(code)
  highlight\_tokens\_some\_other\_way(highlight\_tokens)
}

Further documentation can be found at [https://hexdocs.pm/pearl](https://hexdocs.pm/pearl).

### Feature completeness

[](#feature-completeness)

As far as I can tell, `pearl` can lex all valid Erlang programs. However, I couldn't find a technical specification for Erlang syntax so what I've implemented is based on the Erlang documentation and other Erlang parsers. If there is something missing, please open an issue and I'll implement it as soon as possible.
