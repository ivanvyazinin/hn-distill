# [Unofficial Windows 11 requirements bypass tool now allows you to disable all AI features](/news/unofficial-windows-11-requirements-bypass-tool-now-allows-you-to-disable-all-ai-features/ "Permalink to Unofficial Windows 11 requirements bypass tool now allows you to disable all AI features")

[Usama Jawad](/profile/523233-usama-jawad/) Neowin [@UsamaJawad96](https://www.twitter.com/UsamaJawad96) · Sep 4, 2025 15:46 EDT · Hot! with [20 comments](#comments "View 20 Comments")

![Flyoobe UI](https://cdn.neowin.com/news/images/uploaded/2025/08/1755874627_1280x720-ghost-white-solid-color-background-fotor-20250822195616.webp)

_[Image via Belim (GitHub)](https://github.com/builtbybel/Flyby11/)_

[Flyoobe is a particularly popular third-party tool](https://www.neowin.net/news/unofficial-windows-11-requirements-bypass-tool-gets-a-bunch-of-new-features/) that allows you to customize your Windows 11 installation by removing bloat through the out-of-the-box experience (OOBE). The tool originally started off as a way to [bypass Windows 11 system requirements on unsupported PCs through the "Flyby11" moniker](https://www.neowin.net/news/popular-tool-for-bypassing-windows-11-requirements-check-gets-rebrand-and-new-oobe-features/) but was later rebranded with a pivot in purpose, described above.

In the past few weeks, Flyoobe has received numerous updates, with the previous version, that is 1.6, [bringing a smarter "debloater" and updated OOBE views](https://www.neowin.net/news/windows-11-requirements-bypass-app-gets-smarter-bloat-remover-and-updated-views/). The developer has recently released version 1.7, along with another update to fix some bugs.

Starting off with version 1.7, people who hate the way Microsoft has been stuffing AI features into Windows 11 will be pleased to know that there is an OOBE view that allows you to discover and disable all AI and Copilot features after the installation of the OS. Moreover, the OOBE view that handles bloat removal has been enhanced too, and now allows presets ranging from Minimal to Full, along with the ability to load custom presets from GitHub.

Furthermore, the "Backup installed drivers" feature has been improved to support export to custom folders. In addition, the developer has fixed a critical High DPI bug, made changes to the UI, and refactored the overall code.

Shortly after this release, version 1.7.284 was released as a hotfix, the change log can be viewed below:

*   Fixed a critical bug in the ESU Enrollment package where the selected parameter wasn’t actually being executed. You can now pass the correct parameter for local or online accounts directly, just as documented in the ConsumerESU script.
*   The brand-new OOBE page for exploring and disabling AI in Windows 11 (first introduced yesterday) has been reworked at its core. It now digs even deeper to uncover the ways AI is embedded throughout Windows 11

Flyoobe 1.7.284 can be downloaded through our dedicated Software stories section [here](https://www.neowin.net/software/flyoobe-17/), but do keep in mind that there is a risk associated with customizing your Windows installation in unintended ways, so only download this software if you trust the developer.
